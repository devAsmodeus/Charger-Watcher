# Аудит политики конфиденциальности Charger Watcher

**Применимый закон:** Закон Республики Беларусь от 07.05.2021 № 99-З «О защите персональных данных» (далее — Закон 99-З), действует с 15 ноября 2021 г.

## 1. Текущий текст команды `/privacy`

Сейчас обработчик `cmd_privacy` (`src/bot/main.py`) выдаёт лаконичное сообщение из ~7 строк: упоминается Telegram ID, список подписок, лог уведомлений, отсутствие передачи третьим лицам, команда `/delete_me` и оговорка о неофициальном характере. Этого недостаточно для соответствия Закону 99-З.

## 2. Gap-анализ

## 3. Drop-in замена для команды `/privacy`

Ниже — текст ровно для команды `/privacy` (на русском, под Telegram MarkdownV1; длина ~2700 символов, лимит Telegram 4096). Оркестратор интегрирует его в `src/bot/main.py` вместо текущего тела `cmd_privacy`.

```
*Политика конфиденциальности*

*Оператор:* физ. лицо — разработчик бота @chargerwatcherbot.
*Правовое основание:* ваше согласие, выраженное началом использования бота (ст. 6 Закона РБ № 99-З).

*Какие данные обрабатываются*
• Telegram ID (tg_id) — для идентификации подписчика.
• Список ваших подписок на станции — чтобы знать, куда слать уведомление.
• Платежные записи Telegram Stars (id транзакции, тариф, срок действия) — для платного тарифа.
• Лог отправленных уведомлений — для дедупликации.
• Одноразовая геолокация по `/nearby` — используется в момент запроса и не сохраняется.

*Что НЕ собирается*
Имя, телефон, e-mail, постоянная история геолокации, контакты, файлы.

*Цели обработки*
Доставка уведомлений о свободных зарядных станциях; администрирование платного тарифа; антифрод и дедупликация.

*Сторонние участники обработки*
• Telegram Messenger Inc. — мессенджер и платёжный механизм Stars.
• Selectel — хостинг (сервер в г. Санкт-Петербург, РФ). Это означает трансграничную передачу для пользователей из РБ; продолжая пользоваться сервисом, вы даёте согласие на такую передачу (ст. 9 Закона № 99-З).
• apigateway.malankabn.by — источник статуса станций (ваши данные туда не передаются).

*Срок хранения*
Пока ваша учётная запись активна. По команде `/delete_me` все ваши данные (подписки, лог, платёжные записи) удаляются немедленно, кроме записей, которые Оператор обязан хранить по закону (например, для бухгалтерской отчётности). При понижении до бесплатного тарифа лишние подписки деактивируются (старейшие — первыми).

*Ваши права (ст. 10–13 Закона № 99-З)*
• Получить копию своих данных — по запросу Оператору.
• Исправить неточные данные — через `/start` и переустановку подписок.
• Удалить все данные — `/delete_me` (необратимо).
• Отозвать согласие — равносильно `/delete_me`.
• Подать жалобу в Национальный центр защиты персональных данных РБ (cpdp.gov.by).

*Безопасность*
Данные передаются по защищённым каналам Telegram. Доступ к БД ограничен Оператором.

*Контакты ответственного лица*
Telegram: [USERNAME_TO_FILL]
E-mail для официальных запросов: [EMAIL_TO_FILL]

Полные тексты: ToS и Refund Policy — `/terms` и `/refund_policy` (ссылки в /start).
```

**Длина drop-in текста:** ~2750 символов с учётом Markdown, влезает в одно сообщение Telegram (лимит 4096).

**Плейсхолдеры, которые Оператор должен заполнить:**

- `[USERNAME_TO_FILL]` — Telegram-username Оператора для связи.
- `[EMAIL_TO_FILL]` — почта для юридически значимых обращений (рекомендуется заводная отдельно от личной).
- Команды `/terms` и `/refund_policy` — после реализации в боте; пока их нет, можно убрать последний абзац.

## 4. Engineering follow-ups, вытекающие из аудита

1. Реализовать выгрузку данных пользователя — команда `/export_me` (CSV/JSON по DM). Без неё право на доступ/перенос (ст. 10, 12) поддерживается только в ручном режиме.
2. Реализовать команду `/refund` (нужна также для Refund Policy).
3. Реализовать команды `/terms` и `/refund_policy`, отдающие ссылки на соответствующие документы.
4. Определить и автоматизировать логику деактивации лишних подписок при истечении платного тарифа (сейчас не реализована).
5. Решить вопрос периода хранения платёжных записей после `/delete_me` (требование Telegram + бухгалтерия) — закрепить срок в политике.
